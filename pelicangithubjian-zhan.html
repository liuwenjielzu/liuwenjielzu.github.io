pelican+github建站

Permalink: 2014-11-11 00:00:00+01:00 by leftcloud in pelican git markdown

Tags：pelican git markdown Slug：tech

花了两三天时间，翻墙看了不少别人写的blog，自己瞎捣腾乐在其中，感觉确实很有意思。本来打算研究生换到计算机能够有自己喜欢的东西去做，但是发现总是事与愿违，碰到很多恶心的人和事，不但帮不了你还在不停的拖着你。现在基本框架算是弄好了，只剩下进一步的充实内容和继续捣腾一些有意思的插件了。

看别人的博客总是会有一些坑在里面，可能是具体的环境不一样或者是不够详细，自己领悟不到，在搭的时候发现几处问题： 1. pelican 以markdown为例，写完md文件以后，很多给出的生成网页命令是make publish/make serve， 我自己在执行的时候发现make publish会把repo下的隐藏文件夹.git给删掉，从而导致你后面用git push的时候会报错，估计是因为和之前的不同导致不能识别对应的repo。 这里我改用了pelican文档里面的pelican content，这个命令保留原来的.git。

2.git 尽量先学习操作git和github，非常优秀的工具。基本命令知道差不多就可以了。

3.disqus 这是一个第三方评论系统，只要是支持这个评论系统的blog，你只要有这一个帐号就可以一招鲜吃遍天，不用你再去注册各式各样的帐号。我觉得这是未来的趋势-精细化越来越明显，只要你一个点做的好就足够了，问题就是你要做的足够好。在弄这个的时候碰到一个很搞笑的问题，我用的模版是bootstrap2，这个模版我随便弄了两篇文章，首页却只显示一篇，而且一开始没看到disqus评论界面，以为没弄成功，花了好长时间徘徊，最后发现点击文章下面的more按钮终于显示了，我彻底无语了。。。. 其实这里很简单，上disqus的官网注册一个帐号然后点击add disqus to yourwebsite设置好你要关联的站点和shortname即可，最后在pelicanconf.py里面加上一条配置即可。

随便敲的，具体细节还需自己去研究，欢迎留言共同探讨！
