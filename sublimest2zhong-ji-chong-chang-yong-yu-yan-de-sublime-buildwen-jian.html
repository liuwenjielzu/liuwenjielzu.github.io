sublime(ST2)中几种常用语言的.sublime-build文件

Permalink: 2014-11-26 00:00:00+01:00 by leftcloud in sublime

最近开始接触sublime text 2编辑器，这又是一款编辑神器，高度可定制，适合用于编辑各种编程语言，通过配置编译文件可以在编辑器界面执行编译命令并查看输出结果>，非常方便，但是配置文件是json格式，自己不会写，网上看到别人写的一些配置，贴过来方便自己将来查看。

C：

{
    "cmd": ["make && ./a.out"],
    "path": "/usr/bin:/usr/local/bin:...",
    "shell": true
}

C++ (via g++):

{
    "cmd": ["g++", "$file", "-o", "$file_base_name", "-I/usr/local/include"],
    "selector": "source.c++",
    "windows": {
       "cmd": ["cl", "/Fo${file_path}", "/O2", "$file"]
    }
}

Java:

{
    "cmd": ["javac", "$file_name", "&&", "java", "$file_base_name"],
    "working_dir": "${project_path:${folder}}",
    "selector": "source.java",
    "shell": true
}

Python:

{
    "cmd": ["python", "-u", "$file"],
    "file_regex": "^[ ]*File \"(...*?)\", line ([0-9]*)",
    "selector": "source.python"
}

Ruby (using RVM):

{
    "cmd": ["~/.rvm/bin/rvm-auto-ruby", "$file"],
    "file_regex": "^(...*?):([0-9]*):?([0-9]*)",
    "selector": "source.ruby"
}

PHP:

{
    "cmd": ["/usr/bin/php", "-l", "$file"], <- Could not just use "php" ?
    "file_regex": "^Parse error: .* in (.*?) on line ([0-9]*)",
    "selector": "source.php"
}

.Net (Windows):

{
    "cmd": ["%WINDIR%\\Microsoft.NET\\Framework\\v4.0.30319\\msbuild", "${project_base_name}.sln"],
    "shell": true,
    "working_dir": "${project_path:${folder}}"
}

Haskell:

{
    "cmd": ["runhaskell", "$file"],
    "file_regex": "^(...*?):([0-9]*):?([0-9]*)",
    "selector": "source.haskell"
}

Pandoc (Markdown to HTML):

{
    "cmd": ["pandoc", "-S", "-s", "-f", "markdown", "-t", "html", "-o", "$file_base_name.html", "$file"],
    "selector": "text.html.markdown"
}

基本把常用的一些语言都加进来了，当然很可能你用的这里面没有，那就需要你自己去网上找了，高手们一般都会把配置贴出来。也有可能这里的配置有的不可以用，或许>会出点问题，如果是这样记得留言，最后再次推荐一下这个编辑器，确实非常霸气。=]
